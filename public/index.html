Database Schema Explorer

These are the live tables in the SQLite database file (recruitment.db).

users Table
  • id (INTEGER PK)
  • name (TEXT)
  • email (TEXT UNIQUE)
  • role (TEXT: candidate/recruiter)
  • created_at (DATETIME)
candidates Table
  • id (INTEGER PK)
  • user_id (INTEGER FK)
  • title (TEXT)
  • skills (TEXT)
  • experience_years (INTEGER)
  • education (TEXT)
  • resume_text (TEXT)
jobs Table
  • id (INTEGER PK)
  • recruiter_id (INTEGER FK)
  • title (TEXT)
  • description (TEXT)
  • required_skills (TEXT)
  • required_experience (INTEGER)
  • location (TEXT)
  • salary (TEXT)
  • status (TEXT)
  • created_at (DATETIME)
applications Table
  • id (INTEGER PK)
  • job_id (INTEGER FK)
  • candidate_id (INTEGER FK)
  • match_score (REAL)
  • match_analysis (TEXT JSON)
  • status (TEXT)
  • applied_at (DATETIME)
